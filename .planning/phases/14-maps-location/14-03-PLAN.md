**3. Create `mobile/components/JobMapMobile.tsx`** — read-only map:
```tsx
import { View, Text, StyleSheet } from 'react-native';
import MapView, { Marker } from 'react-native-maps';
import { palette } from '../app/constants/design';

type Props = {
  latitude: number;
  longitude: number;
  address?: string | null;
};

export default function JobMapMobile({ latitude, longitude, address }: Props) {
  return (
    <View>
      <MapView
        style={styles.map}
        initialRegion={{
          latitude,
          longitude,
          latitudeDelta: 0.01,
          longitudeDelta: 0.01,
        }}
        scrollEnabled={false}
        zoomEnabled={false}
      >
        <Marker coordinate={{ latitude, longitude }} />
      </MapView>
      {address && <Text style={styles.address}>{address}</Text>}
    </View>
  );
}

const styles = StyleSheet.create({
  map: {
    height: 250,
    width: '100%',
    borderRadius: 10,
    overflow: 'hidden',
  },
  address: {
    fontSize: 13,
    color: palette.muted,
    marginTop: 4,
  },
});
```

**4. Create `mobile/components/LocationPickerMobile.tsx`** — address search + confirmation map:
- Props: `{ latitude, longitude, address, onChange }` where `onChange: (loc: { latitude: number | null; longitude: number | null; address: string | null }) => void`
- State: `query` (string), `results` (array of `{ label: string; lat: number; lng: number }`), `loading` (boolean)
- Geocoding: use `fetch` to Nominatim directly (not leaflet-geosearch, which is web-only):
  ```ts
  const url = `https://nominatim.openstreetmap.org/search?format=json&countrycodes=ro&limit=5&q=${encodeURIComponent(query)}`;
  const res = await fetch(url, { headers: { 'User-Agent': 'HandyLink/1.0' } });
  const data = await res.json();
  const mapped = data.map((r: any) => ({ label: r.display_name, lat: parseFloat(r.lat), lng: parseFloat(r.lon) }));
  ```
  Nominatim requires a User-Agent header per usage policy.
- Debounce: 400ms timeout, only when `query.length >= 3`
- Render structure (using `View`, `TextInput`, `FlatList`, `TouchableOpacity`, `ActivityIndicator`):
  1. Label: `<Text style={styles.label}>Job Location (optional)</Text>`
  2. Search input: `<View>` with `TextInput` styled with `borderWidth: 1, borderColor: palette.border, borderRadius: 10, padding: 12, fontSize: 16`. Placeholder: `"Search address, e.g. Strada Florilor, Cluj"`. When loading, show `ActivityIndicator` positioned absolute right
  3. Results list: `FlatList` with up to 5 items, each a `TouchableOpacity` showing `item.label`. On press: call `onChange({ latitude: item.lat, longitude: item.lng, address: item.label })`, clear results, set query to item.label
  4. Confirmation map: when `latitude && longitude`, render `<View style={{ marginTop: 8 }}><MapView ...>` with same styling as JobMapMobile but `scrollEnabled={true}`. Pin NOT draggable on mobile (per UI-SPEC: "not draggable on mobile (address-only input)")
  5. Clear link: when latitude set, `<TouchableOpacity onPress={() => { onChange({ latitude: null, longitude: null, address: null }); setQuery(''); }}><Text style={{ color: '#dc2626', fontSize: 13, marginTop: 4 }}>Remove location</Text></TouchableOpacity>`
- Error state: if fetch fails or returns empty, show `<Text style={{ color: palette.muted, fontSize: 13, marginTop: 4 }}>No results found. Try a different address or skip this field.</Text>`
- Style labels with `fontSize: 13, fontWeight: '600', color: palette.text, marginBottom: 4` matching existing post-job.tsx label style

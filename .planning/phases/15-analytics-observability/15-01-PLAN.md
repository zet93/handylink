  <action>
**Program.cs modification:**

After line `builder.WebHost.UseUrls(...)` and before `Stripe.StripeConfiguration.ApiKey = ...`, insert:

```csharp
builder.WebHost.UseSentry(o =>
{
    o.Dsn = builder.Configuration["Sentry:Dsn"];
    o.TracesSampleRate = 0.1;
    o.SendDefaultPii = false;
});
```

`SendDefaultPii = false` is mandatory — prevents email and IP from being sent to Sentry (GDPR).
`TracesSampleRate = 0.1` captures 10% of traces, matching the rate set on web and mobile.

**appsettings.json modification:**

Add a `Sentry` section alongside the existing top-level keys:

```json
"Sentry": {
  "Dsn": ""
}
```

The empty string is intentional — real DSN value goes in Render environment variables as `Sentry__Dsn` (double underscore = colon in .NET config hierarchy) or in user secrets via:
```bash
dotnet user-secrets set "Sentry:Dsn" "https://xxx@oyyy.ingest.sentry.io/zzz"
```

After editing, verify the API still builds:
```bash
dotnet build backend/HandyLink.API
```

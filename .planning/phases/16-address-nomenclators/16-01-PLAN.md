NOTE: RESEARCH.md Data Source section shows a nested derivation format (array of county objects each containing a `cities` string array). This plan uses the FLAT shape as specified in 16-UI-SPEC.md Data Shape section: a top-level `counties` array plus a top-level `cities` array with `county_id` references. The flat shape is the canonical output. Disregard the nested format shown in RESEARCH.md derivation steps.

Build ro-nomenclator.json using this exact shape (per 16-UI-SPEC.md Data Shape section):

```json
{
  "counties": [
    { "id": "AB", "name": "Alba" },
    ...
  ],
  "cities": [
    { "id": "AB-001", "county_id": "AB", "name": "Alba Iulia" },
    ...
  ]
}
```

Derivation steps:
1. Fetch https://raw.githubusercontent.com/virgil-av/judet-oras-localitati-romania/master/judete.json via WebFetch or construct from the verified structure in RESEARCH.md.
2. For each county entry in that source (fields: "auto", "nume", "localitati"):
   - Map "auto" → county "id"
   - Map "nume" → county "name" (keep Romanian diacritics — use "nume" field, NOT "simplu")
   - Filter "localitati" to entries that have NO "comuna" field (these are city/town tier, not villages)
   - For each filtered localitate, create a city entry: `{ "id": "{AUTO}-{padded_index}", "county_id": "{AUTO}", "name": "{localitate.nume}" }`
3. Hand-author București special case:
   - County entry: `{ "id": "B", "name": "București" }`
   - Six city entries: `{ "id": "B-S1", "county_id": "B", "name": "Sector 1" }` through `{ "id": "B-S6", "county_id": "B", "name": "Sector 6" }`
   - The virgil-av source may already have a "B" entry; if so, replace its localitati with the 6 Sector entries only.
4. Ensure all 42 counties are present (AB, AR, AG, BC, BH, BN, BT, BV, BR, B, BZ, CS, CL, CJ, CT, CV, DB, DJ, GL, GR, GJ, HR, HD, IL, IS, IF, MM, MH, MS, NT, OT, PH, SM, SJ, SB, SV, TR, TM, TL, VS, VL, VN — 41 counties + B = 42).
5. Write the completed JSON to frontend/src/data/ro-nomenclator.json.
6. Copy the identical file to mobile/assets/ro-nomenclator.json.

Then create frontend/src/data/__tests__/ro-nomenclator.test.js:

```js
import { describe, it, expect } from 'vitest';
import nomenclator from '../ro-nomenclator.json';

describe('ro-nomenclator.json', () => {
  it('NOM-05: has exactly 42 county entries', () => {
    expect(nomenclator.counties).toHaveLength(42);
  });

  it('NOM-06: București county has exactly 6 sector cities', () => {
    const bCities = nomenclator.cities.filter(c => c.county_id === 'B');
    expect(bCities).toHaveLength(6);
    const names = bCities.map(c => c.name);
    expect(names).toContain('Sector 1');
    expect(names).toContain('Sector 6');
  });

  it('every county entry has id, name fields', () => {
    nomenclator.counties.forEach(county => {
      expect(county).toHaveProperty('id');
      expect(county).toHaveProperty('name');
    });
  });

  it('every city entry has id, county_id, name fields', () => {
    nomenclator.cities.forEach(city => {
      expect(city).toHaveProperty('id');
      expect(city).toHaveProperty('county_id');
      expect(city).toHaveProperty('name');
    });
  });

  it('all city county_id values reference a valid county', () => {
    const countyIds = new Set(nomenclator.counties.map(c => c.id));
    nomenclator.cities.forEach(city => {
      expect(countyIds.has(city.county_id)).toBe(true);
    });
  });
});
```

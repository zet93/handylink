| Boundary | Description |
|----------|-------------|
| Client request body → API | County string arrives in CreateJobDto / UpdateUserDto from untrusted HTTP body |
| JWT → GetUserId() | User identity always from JWT, never from request body |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-05 | Tampering | CreateJobDto.County | mitigate | EF Core parameterized queries prevent SQL injection; county is a nullable TEXT column with no privilege implications |
| T-16-06 | Tampering | UpdateUserDto.County | mitigate | Same — parameterized EF Core query; `if (dto.County is not null)` guard prevents empty-string overwrite |
| T-16-07 | Elevation of Privilege | JobsController user identity | accept | GetUserId() always reads from JWT (BaseController pattern); County in request body has no auth implication |
| T-16-08 | Input Validation | County/City string values | mitigate | Server-side: county is optional string (nullable); no length exploit possible on TEXT column via EF Core; frontend validates via Zod (NOM-05) |
</threat_model>

## Interaction Flow

### Web — County → City cascade

1. User sees County `<select>` with placeholder option "Selectează județul…" and City `<select>` disabled with "Selectează județul întâi".
2. User picks a county → City `<select>` is enabled and repopulates with cities for that county.
3. User picks a city → form field values set via React Hook Form `setValue('county', ...)` and `setValue('city', ...)`.
4. On PostJobPage only: after city is set, trigger geocode via Nominatim for that city name (city + ", Romania") and call LocationPicker's `onChange` to auto-center the map (D-05). The map pin can then be further adjusted by the user.
5. Validation: both county and city are required. Error shown below each `<select>` using `text-red-500 text-xs mt-1` pattern.

### Mobile — County → City modal cascade

1. User sees "Județ" trigger field (unfilled, placeholder "Selectează județul…") and "Oraș / Comună" trigger field (disabled, placeholder "Selectează județul întâi").
2. User taps County field → full-screen modal slides up with FlatList of all 41 counties + Bucharest.
3. User taps a county → modal closes, county field shows selected county name, city field becomes enabled.
4. User taps City field → full-screen modal slides up with cities filtered to selected county.
5. User taps a city → modal closes, city field shows selected city name.
6. On post-job screen only: after city is set, trigger Nominatim geocode and call `LocationPickerMobile`'s `onChange` to auto-center the map (D-05).
7. Validation before submit: if county or city is empty, `Alert.alert('Eroare', 'Județul și orașul sunt obligatorii.')`.

### Bucharest Special Case

Bucharest appears as a county-level entry named "București" (separate from "Ilfov"). Its cities list contains: "Sector 1", "Sector 2", "Sector 3", "Sector 4", "Sector 5", "Sector 6". This is the discretionary choice for the CONTEXT.md note about Bucharest sectors.

## Data Shape

The bundled JSON (`ro-nomenclator.json`) uses a flat list with county reference:

```json
{
  "counties": [
    { "id": "AB", "name": "Alba" },
    { "id": "B", "name": "București" },
    ...
  ],
  "cities": [
    { "id": "AB-001", "county_id": "AB", "name": "Alba Iulia" },
    { "id": "B-S1", "county_id": "B", "name": "Sector 1" },
    ...
  ]
}
```

City names: Romanian diacritics preserved (e.g. "Timișoara", "Cluj-Napoca", "Târgu Mureș") — this is the discretionary choice from CONTEXT.md. The SIRUTA source uses diacritics; preserving them matches Romanian government form conventions.

County names: Displayed as full Romanian name with diacritics (not abbreviations) in the dropdown/modal.

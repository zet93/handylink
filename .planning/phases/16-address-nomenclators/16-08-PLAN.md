  <action>
Add a MapRecenter child component inside LocationPicker that calls map.flyTo() when latitude/longitude props change.

The fix: inside LocationPicker.jsx, add a new inner component MapRecenter that uses useMap() from react-leaflet. Add it to the imports from react-leaflet alongside Marker. Place it as a child of MapContainer (which is currently inside JobMap, a separate component).

The problem is that JobMap is a separate component that owns the MapContainer, so we cannot easily inject a child into it from LocationPicker without modifying JobMap. Two options:

Option A (modify LocationPicker to not use JobMap):
- Remove the use of JobMap in LocationPicker
- Directly render the MapContainer + TileLayer + DraggableMarker in LocationPicker
- Add a <MapRecenter lat={latitude} lng={longitude} /> child inside MapContainer

Option B (modify JobMap to accept a flyTo trigger):
- Too invasive; JobMap is also used in read-only job detail pages

Use Option A — LocationPicker already has all the state needed. The leaflet CSS import and icon fix from JobMap.jsx must be preserved (copy them into LocationPicker.jsx or import JobMap is removed and the css/icon setup stays global via a separate import).

Concrete changes to LocationPicker.jsx:

1. Add import: `import { MapContainer, TileLayer, useMap } from 'react-leaflet';`
2. Keep the existing `import { Marker } from 'react-leaflet';` — merge both into one import line
3. Add import at top: `import 'leaflet/dist/leaflet.css';`
4. Add import for leaflet icon fix (same as JobMap.jsx):
   ```
   import L from 'leaflet';
   import markerIcon2x from 'leaflet/dist/images/marker-icon-2x.png';
   import markerIcon from 'leaflet/dist/images/marker-icon.png';
   import markerShadow from 'leaflet/dist/images/marker-shadow.png';
   delete L.Icon.Default.prototype._getIconUrl;
   L.Icon.Default.mergeOptions({ iconRetinaUrl: markerIcon2x, iconUrl: markerIcon, shadowUrl: markerShadow });
   ```
5. Add inner component before the main export:
   ```javascript
   function MapRecenter({ lat, lng }) {
     const map = useMap();
     useEffect(() => {
       if (lat && lng) map.flyTo([lat, lng], 14);
     }, [lat, lng, map]);
     return null;
   }
   ```
6. Replace the `{latitude && longitude && (<div className="mt-2"><JobMap ... /></div>)}` block with:
   ```jsx
   {latitude && longitude && (
     <div className="mt-2">
       <div className="h-64 w-full rounded-lg overflow-hidden border border-gray-200">
         <MapContainer
           center={[latitude, longitude]}
           zoom={14}
           style={{ height: '100%', width: '100%' }}
           scrollWheelZoom={false}
         >
           <TileLayer
             url="https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png"
             attribution='&copy; <a href="https://www.openstreetmap.org/copyright">OpenStreetMap</a> contributors'
           />
           <DraggableMarker latitude={latitude} longitude={longitude} address={address} onChange={onChange} />
           <MapRecenter lat={latitude} lng={longitude} />
         </MapContainer>
       </div>
       {address && <p className="text-xs text-gray-500 mt-1">{address}</p>}
     </div>
   )}
   ```
7. Remove the `import JobMap from './JobMap';` line

Note: The leaflet CSS import in LocationPicker may cause a duplicate CSS load (JobMap.jsx also imports it). This is harmless — CSS imported twice is de-duped by the bundler.

```

setValue('city', e.target.value, { shouldDirty: true, shouldTouch: true });
```

No other changes needed. The `{...register('county')}` spread + custom `onChange` override pattern stays as-is.

  <behavior>
    - When `signInWithGoogle()` is called on web/mobile, user is authenticated via Supabase OAuth.
    - If this is first login and no profile exists, user is rerouted to role selection, then profile row is inserted with role and optional worker_profiles row.

    - Existing email/password account with same email is recognized and not duplicated; returned user id is same.
  </behavior>
  <action>

       - Add `signInWithGoogle = () => supabase.auth.signInWithOAuth({ provider: 'google', options: { redirectTo: window.location.origin + '/auth-callback' }})`.
       - Add helper to fetch `GET /api/users/me` after OAuth session is ready.
       - Implement role onboarding function `completeRoleSelection(role)` that posts to backend update and sets context state.
    2. In `mobile/services/supabase.ts`: ensure `createClient` has `auth.flowType = 'pkce'` already and add helper `signInWithGoogle` to call `supabase.auth.signInWithOAuth({ provider: 'google' })`.
    3. In `backend/HandyLink.Core/Services/UserService.cs`: add `EnsureUserProfileAsync(Guid userId, string role)` that upserts profile and returns user profile.
       - If user exists: preserve existing role and data.
       - If missing: create profile with role. if role is `worker` or `both`, create `worker_profiles` row as needed.
    4. In `backend/HandyLink.API/Controllers/UsersController.cs`: add endpoint `POST /api/users/me/role` to call service.
  </action>
